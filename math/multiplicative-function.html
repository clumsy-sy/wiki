积性函数

对于数论函数 f(x)​，若对于任意的 a,b​ 满足 a⊥b​​（即 a,b 互质） 都有

f(ab)=f(a)f(b)

则称 f(x) 为积性函数。

若对任意的 a,b 都有 f(ab)=f(a)f(b)，则称 f(x) 为完全积性函数。

例子

积性函数的例子有

  • Euler 函数：φ(n):=i=1∑n​[gcd(i,n)=1]。
  • Mobius 函数：μ(n):=⎩⎨⎧​10(−1)ω(n)​n=1∃d>1,d2∣notherwise​

完全积性函数的例子有

  • 单位函数：ε(x):=[x=1]。
  • 指数函数：idk​(x):=xk。
  • 常数函数：1(x):=1​。

性质

积性函数与积性函数的卷积是积性函数，积性函数对卷积的逆元也是积性函数，即积性函数的全体构成了一个 Abel 群，其单位元是 ε​。

快速求前缀和，杜教筛 是一种可选的方案。但是其该算法有一些局限性，下面给出两种适用更广泛的快速计算方法。

把质数集记作 x∈/P，第 i 个质数记作 pi​。特殊的，p0​=1。

记 x 的最小非 1 因子为 lpf(x)，显然也是质因子。容易发现，若 x∈/P，则有 lpf(x)⩽x​。

Min_25 筛

Min_25 提出了 “Min_25 筛” 。之后，Min_25 还进行了改进，想了解的可以自行了解，这里我们只讲朴素 Min_25 筛。

记

F(n,k)=x=2∑n​[pk​<lpf(x)]f(x)

和

h(n)=2⩽p⩽n∑​f(p)

于是 S(n) 可以表示为

i=1∑n​f(i)=f(1)+F(n,0)

接下来对各个部分分别求解。

Part 1

类似的手法，处理 F(n,k) 函数的计算。

质数单独计算贡献，合数则枚举最小因子 pi​，可以得到 F(n,k) 的递推式

F(n,k)​=pk​<pi​⩽n​∑​pic​⩽n∑​f(pic​)⎝⎛​[c>1]+x=2∑⌊n/pic​⌋​[pi​<lpf(x)]f(x)⎠⎞​+pk​<p⩽n∑​f(p)=pk​<pi​⩽n​∑​pic​⩽n∑​f(pic​)([c>1]+F(⌊pic​n​⌋,i))+h(n)−h(pk​)​

再注意到，若 c 满足 pic​⩽n<pic+1​，可以得到

⌊pic​n​⌋<pi​<pi+1​⟹F(⌊pic​n​⌋,i)=0

而当 c=1 时第一项 [c>1]=0，因此可以错开有

F(n,k)=pk​<pi​⩽n​∑​pic+1​⩽n∑​(f(pic​)g(⌊pic​n​⌋,i)+f(pic+1​))+h(n)−h(pi​)

如上所述，我们可以写出递归代码。取模有时可以用 int128 优化。

ll F(ll n, int k) {
    if(n <= prime[k])
        return 0;
    ll ret = s[id(n)] - s[prime[k]];
    for (int i = k + 1; i <= cnt && prime[i] * prime[i] <= n; i++) {
        ll pi = prime[i], pk = pi;
        for (int c = 1; pk * pi <= n; c++, pk *= pi)
            ret += f_p(pi, c) * F(n / pk, i) + f_p(pi, c + 1);
    }
    return mo(ret);
}

Part 2

接下来，关键在于求 h。预处理 [1,n] 所有的 h 不但时间上不够，空间上也不够。注意到我们所有用到的 h(x) 都是 F(n,0) 递归产生的，分类讨论：

  • 对于递归产生的 h(pi​)，总是有 pi​⩽n​。
  • 对于递归产生的 h(n)，都是通过 g 的递推得到的，是 n 的不断整除。又有定理

⌊c⌊a/b⌋​⌋=⌊bca​⌋

这启发我们用整数分块，精简大于 n​ 的预处理。

我们只考虑 f(p) 是较低次数的多项式的情况，令 gs​(p)=ps（这样凑出来的 g 是完全积性函数），单独计算每一项后求和。

再联想到 Eratosthenes 筛法：每次枚举一个 p，筛去所有不小于 p2 的 p 的倍数。设第 k 论筛完后剩下的数的 g(p) 之和为

Gk​(n)=i=2∑n​[pk​<lpf(i)∨i∈P]g(i)

容易发现，当 n<pk2​ 时，Gk​(n)=Gk−1​(n)。当 n⩾pi2​ 时，递推有

Gk​(n)=Gk−1​(n)−g(pk​)(Gk−1​(⌊pk​n​⌋)−Gi−1​(pk​−1))

至此，我们完成了朴素 Min_25 筛的全部推导。

LOJ 6053 简单的函数

定义 f(pc)=p⊕c。

因此 g(p)=f(p)=p−1+2[p=2]。与筛 φ 操作一样，最后加上 2 即可。

const ll mod = 1000000007, inv2 = 500000004;
const int maxn = 200005;
ll n, prime[maxn], cnt, w[maxn], s[maxn], c[maxn];
ll sqrt_n, m;

ll mo(ll x) {
    return (x + mod) % mod;
}

ll f_p(ll p, ll e) {
    return p ^ e;
}

int id(ll x) {
    return x <= sqrt_n ? x : m - (n / x) + 1;
}

ll F(ll n, int k) {
    if(n <= prime[k])
        return 0;
    ll ret = s[id(n)] - s[prime[k]];
    for (int i = k + 1; i <= cnt && prime[i] * prime[i] <= n; i++) {
        ll pi = prime[i], pk = pi;
        for (int c = 1; pk * pi <= n; c++, pk *= pi)
            ret += f_p(pi, c) * F(n / pk, i) + f_p(pi, c + 1);
    }
    return mo(ret);
}

int main() {
    n = rr();
    sqrt_n = sqrt(n + 0.01);
    m = cnt = 0;
    for (ll l = 1, r; l <= n; l = r + 1) {
        w[++m] = r = n / (n / l);
        ll mr = mo(r);
        s[m] = mr * (mr + 1) / 2 - 1;
        c[m] = mr - 1;
    }
    for (ll p = 2; p <= sqrt_n; p++) {
        if(c[p] != c[p - 1]) {
            prime[++cnt] = p;
            for (ll j = m; w[j] >= p * p; j--) {
                s[j] -= p * (s[id(w[j] / p)] - s[p - 1]);
                c[j] -= c[id(w[j] / p)] - c[p - 1];
            }
        }
    }
    for (int i = 2; i <= m; i++)
        s[i] = mo(s[i] - c[i] + 2);
    printf("%lld", F(n, 0) + 1);
}

P4213 杜教筛

求 Euler φ 函数的前缀和，有 φ(pk)=pk−pk−1。

令 g(p)=p，那么 G0​(n)=21​n(n+1)−1。令 g(p)=1，那么 G0​(n)=n−1。

两次计算分别累加即可。

求 Mobius μ 函数的前缀和，有 μ(pk)=−[k=1]，令 g(p)=1，在筛 φ(n) 中已经计算过所以不用再次计算。

再由于当 c>1 时有 μ(pc)=0，故计算 F 时可以再优化。在计算时要注意卡常，避免除法，最后一个点对时间的要求比较紧。

舟阁筛

设 n 的唯一分解是

n=p1k1​​p2k2​​⋯psks​​

则积性函数 f(x)​​​ 的在 x=n 处的值有

f(n)=f(p1k1​​)f(p2k2​​)⋯f(psks​​)

由于数 i 最多只能拥有一个大于 n​​ 的因子，因此按照是否具有分类，可以得到

i=1∑n​f(i)​=x⩽nx 无大于 n​ 的质因子​∑​⎣⎡​f(x)⎝⎛​1+n​<p⩽⌊n/x⌋∑​f(p)⎠⎞​⎦⎤​=x=1∑n​​f(x)p>n​∑⌊n/x⌋​f(p)+x⩽nx 无大于 n​ 的质因子​∑​f(x)​

关键就在如何快速求这几部分。

Part 1

考虑记 [1,l] 中不含因子 p1​,⋯,pt​ 的数在 f 下取值之和为 g(t,l)。递推关系有

g(t,j)=g(t−1,j)−f(pt​)g(t−1,⌊l/pt​⌋)

边界为 g(0,l)=i=1∑l​f(i)​。设 [1,n​]​ 内质数的个数为 m，因此第一部分即化为计算

x=1∑n​​f(x)p>n​∑⌊n/x⌋​f(p)=x=1∑n​​f(x)g(m,⌊n/x⌋)

然而朴素的预处理 g 的复杂度是 O(lognn​)​，需要再优化。

容易发现当 pt+1​>l 时有 g(t,l)=f(1)=1。故当 pt2​>l 时，代入递推有

g(t,l)=g(t−1,j)−f(pt​)

因此 pt2​>j​ 时的计算是可以省去的，直接累加这一段质数取值即可。此时复杂度 O(lognn43​​)​​。

Part 2

再考虑记 [1,l] 中只含因子 pt​,⋯,pm​ 的数在 f​ 下取值之和为 h(t,l)​。递推有

h(t,l)=h(t+1,l)+c=1∑​f(ptc​)h(t+1,⌊l/ptc​⌋)

其边界条件为 h(m+1,l)=f(1)=1​。于是

x⩽nx 无大于 n​ 的质因子​∑​f(x)=h(0,n)

类似的优化 h 的计算。当 pt​>l 时有 h(t,l)=f(1)=1，代入递推，当 pt2​>l 时有

h(t,l)=h(t−1,l)+f(pt​)

同样直接累加。此处复杂度 O(lognn43​​)​​。

All

总复杂度 O(lognn43​​)。我们要求 f(p)​ 是容易计算的，否则其会增加额外的损耗。

因为出现了更好用的 Min_25 筛，舟阁筛已经用不到了。
